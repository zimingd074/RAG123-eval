# 「小米生态商品知识库」文档清单反推设计

> **业务背景**：知识库服务于小米商城智能客服，商品命名采用小米、Redmi、米家、小爱及生态链品牌语义。
>
> **政策声明**：售后政策库中的保修、退换货、价保、物流、发票和会员内容均为学习与评测用途的「小米商城模拟售后政策」，不代表小米官方真实政策；真实权益以小米官方渠道为准。
>
> **设计原则**：
>
> 1. **以评估集 Query 为锚**：每条 Query 必须能在文档库中找到答案
> 2. **文档颗粒度合理**：不过粗（影响检索精度）、不过细（影响维护成本）
> 3. **真实可仿造**：参考小米商城真实文档结构
> 4. **难点可埋**：刻意设计参数表、对比表、故障树等结构化内容
>

---

## 架构边界

- 静态商品参数、使用手册、配网说明、故障排查与模拟售后政策进入知识库，通过 RAG 检索。
- 动态价格、库存、推荐、商品对比与实时活动不维护静态答案，统一通过 Tool Calling 获取或计算。
- 不维护商品两两对比文档，避免商品组合爆炸；S3 对比选购由 `compare_products` 等工具动态完成。
