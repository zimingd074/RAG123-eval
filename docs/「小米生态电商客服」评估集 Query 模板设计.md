# 「小米生态电商客服」评估集 Query 模板设计

> **业务背景**：面向小米商城与小米生态商品的 RAG 智能客服。购物、订单、地址、发票和会员场景使用「小米商城 APP」；设备添加、智能家居控制和自动化联动场景使用「米家 APP」。
>
> **政策声明**：本文中的保修、退换货、价保、物流、发票和会员规则均为学习与评测用途的模拟数据，不代表小米官方真实政策；真实权益以小米官方渠道为准。
>
> **目标**：为 22 个意图各设计 5-10 条 Query，构建一份 ~150 条的评估集
>
> **设计原则**：
>
> 1. **难度分层**：每个意图含 简单 / 中等 / 困难 三档
> 2. **真实口语化**：模拟真实用户输入（含错别字、不完整、口语）
> 3. **难点预埋**：刻意设计参数歧义、多跳推理、拒答陷阱等
> 4. **可扩展**：后续可基于模板用 LLM 扩展到 500+ 条
>
